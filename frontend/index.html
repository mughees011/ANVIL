Core loop: Plan → Execute → Quality Check → Self-Heal (retry or re-plan)
◫ Component Reference
Planner
Takes a Task + available Tools + Memory → produces a structured Plan (ordered Steps with dependencies).
Executor
Runs one Step at a time, resolves Tool(s), calls them via GroqClient, captures results.
QualityEnforcer
Rule-based checks (always) + optional LLM-graded rubric checks. Fast-fail then fuzzy-grade.
SelfHealingEngine
On failure: same-step retry for exceptions, partial re-plan for quality failures. Max 3 retries default.
